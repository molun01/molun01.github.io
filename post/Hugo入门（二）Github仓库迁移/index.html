Hugo入门（二）Github仓库迁移

2019-10-22 #博客

GitHub Pages规则更新，将代码仓库迁移可以更灵活的在Github上部署博客。

迁移后的好处

可以从不同仓库gh-pages分支部署博客了，方便切换主题或控制版本

如何迁移

新建一个仓库

  1. 在此页面的Repository name框中，为您的代码库起一个名字，例如:Blog。
  2. 添加自述文件README.md到其中。（可选）
  3. 单击Create repository，创建成功可看到三种添加文件方式的指导;

将您的文件上传到GitHub

*如果是搭建一个全新的博客可参考Hugo入门（零）

迁移步骤

  1. 新建好仓库后，选择第三个添加文件的方式即从其他仓库中导入项目，挺慢的，成功的标志为邮件通知。
  2. 现在你需要为你的仓库创建一个gh-pages分支，该分支会在特殊位置发布。
  3. 它的URL采用username.github.io/Repository-name的格式，所以在我的例子中，URL是https://molun01.github.io/Blog。显示的页面是index.html页面。

测试博客

在新的浏览器标签中浏览您的GitHub Pages的网址，您将能够在线访问您的网站！通过电子邮件把地址发送给你的朋友，炫耀你的掌握的成果吧。
注意 ：如果卡住了，GitHub Pages也是非常有帮助的。

更多的GitHub知识

如果您想对test-site进行更多更改并将其上传到GitHub，那么您只需像以前一样对文件进行更改。然后，您需要输入以下命令（在每个之后按Enter键）将这些更改推送到GitHub：

$ git add -A
$ git commit -m 'another commit'
$ git push

您可以使用更合适的消息替换上一次的提交信息，以描述您刚刚做出的更改。

我们仅仅提供了Git的浅显基本的信息。要了解更多信息，请先从GitHub帮助站点开始。
